# Phase 5 Acceptance Criteria

## Functional Acceptance
- [ ] User can complete full flow: login -> start session -> answer -> submit -> view results.
- [ ] User can view and update profile including grade.
- [ ] Progress screen displays real data from backend.

## Quality Acceptance
- [ ] All P0 backlog items complete.
- [ ] No P0/P1 UI or integration bug in core flow.
- [ ] Core screens include loading/error/empty handling.

## Reliability Acceptance
- [ ] Session recovery after app interruption works.
- [ ] Submit behavior is idempotent from user perspective.
- [ ] Crash-free rate meets beta readiness target.

## Platform Acceptance
- [ ] Android staging build passes QA checklist.
- [ ] iOS staging build passes QA checklist.

## Exit Decision
Phase 5 is complete when all acceptance criteria and mandatory test matrix items are checked.
