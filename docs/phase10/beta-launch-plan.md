# Phase 10 Beta Launch Plan

## 1. Beta Objectives
- Validate core learning flow in real-world usage.
- Measure stability and engagement KPIs.
- Identify high-impact bugs and UX friction before wider release.

## 2. Audience and Cohort
- Initial cohort size: 20-50 testers.
- Mix recommendation:
  - students across multiple grades
  - parents/guardians
  - internal QA/stakeholders

## 3. Rollout Stages
1. Internal beta (team + trusted testers)
2. Closed beta (selected external users)
3. Expanded beta (if metrics stable)

## 4. Build Strategy
- Android: Play Console Internal/Closed track.
- iOS: TestFlight Internal/External testers.
- Tag each release build with changelog and known issues.

## 5. Beta Duration
- Suggested initial window: 2-4 weeks.
- Weekly release cadence for fixes and UX improvements.

## 6. Success Criteria (example)
- Crash-free sessions above target.
- Session completion rate above target.
- No unresolved P0 defects for sustained period.
- Positive trend in repeat usage and streak activity.
