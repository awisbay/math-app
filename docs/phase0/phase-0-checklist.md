# Phase 0 Checklist

## Goal
Lock scope, architecture, repo baseline, and delivery plan before feature development.

## Checklist
- [x] Confirm MVP scope and explicitly mark out-of-scope items.
- [x] Confirm target users for beta (students + parents) and age range.
- [x] Confirm core app rules:
  - [x] Session duration is exactly 15 minutes (900 seconds).
  - [x] Session has exactly 10 questions.
  - [x] Language is Indonesian only for MVP.
  - [x] Grade support is Kelas 1 SD to Kelas 12 SMA.
  - [x] Streak increments when user completes at least 1 session/day.
- [x] Confirm monorepo strategy for MVP.
- [x] Setup monorepo directory skeleton.
- [x] Lock technology choices (FE, BE, DB, auth, hosting, analytics).
- [x] Write architecture decisions and tradeoffs.
- [x] Setup coding standards and branch strategy.
- [x] Setup CI minimum pipeline (lint + test smoke + build check).
- [x] Create initial ticket backlog for Phase 1 and Phase 2.
- [ ] Define beta timeline and KPI targets.

## Exit Criteria
- [x] Scope and architecture are approved.
- [x] Repo structure is ready for implementation.
- [x] Baseline CI is running on pull requests.
- [x] Backlog is prioritized for the next 2 phases.
